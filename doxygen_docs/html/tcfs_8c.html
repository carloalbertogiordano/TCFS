Get the size of a file in the TCFS file system.

This function is called when the getattr operation is performed on a file in the TCFS to obtain file attributes.

Parameters
fuse_pathThe path of the encrypted file for which the size is requested.
stbufBuffer to store the file attributes, including the size.
fiFile information structure provided by FUSE.
Returns
TCFS_SUCCESS on success, negative error code on failure.

The tcfs_file_size function is invoked to retrieve the size of a file within the TCFS. It decodes the encrypted file path, translates it into the actual file path on the underlying file system, and then uses the getattr function to obtain the file attributes, including the file size.

Parameters
fuse_pathThe path of the encrypted file within the TCFS.
stbufBuffer to store the file attributes, including the file size.
fiFile information provided by FUSE, which may be used to obtain additional details about the file if needed.
Returns
true on success. On failure, it returns a negative error code representing the type of error encountered.
Note
  • The function is a crucial part of file attribute retrieval, and the size is a fundamental attribute of a file.
  • The correct functioning of this function is essential for providing accurate information about the file size.
Warning
  • Ensure that the function correctly translates the encrypted file path into the actual file path on the underlying file system.
  • Verify that the file attributes, especially the size, are accurately retrieved and reported in the stbuf buffer.

Modify the access and modification timestamps of a file in the TCFS file system.

This function is called when the utimens operation is performed on a file in the TCFS.

Parameters
fuse_pathThe path of the encrypted file for which timestamps need to be modified.
tsAn array of two timespec structures containing the new access and modification timestamps.
fiFile information structure provided by FUSE.
Returns
TCFS_SUCCESS on success, negative error code on failure.

The tcfs_utimens function is invoked to modify the access and modification timestamps of a file within the TCFS. It decodes the encrypted file path, translates it into the actual file path on the underlying file system, and then uses the utimes function to apply the changes to the file timestamps.

Parameters
fuse_pathThe path of the encrypted file within the TCFS.
tsAn array containing two timespec structures. The first structure represents the new access timestamp, and the second represents the new modification timestamp.
fiFile information provided by FUSE, which may be used to obtain additional details about the file if needed.
Returns
true on success. On failure, it returns a negative error code representing the type of error encountered.
